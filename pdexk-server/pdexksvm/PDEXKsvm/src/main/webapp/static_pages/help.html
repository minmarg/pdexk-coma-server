PDEXK server HELP
The PDEXK server is for detecting new protein families of the PDEXK superfamily using HHsearch profile-profile alignments and Support Vector Machines. A short description of server input options and output is given below.

Server input
Input

Paste protein sequence or multiple alignment
Paste a single protein sequence or multiple alignment in the format defined by the 'Input type'. Currently, only the FASTA format is accepted. If a multiple sequence alignment is provided, the length of aligned sequences must be the same.

Upload a local file
Optionally, user may upload a local file containing a single protein sequence or multiple sequence alignment in the format defined by the 'Input type'.

Input type
Describes the format of input protein sequence(s). Currently, The FASTA format is available. Each sequence in this format has a description line starting with symbol '>', e.g.:
>3i1m_R mol:protein length:75 30S ribosomal protein S18
MARYFRRRKFCRFTAEGVQEIDYKDIATLKNYITESGKIVPSRITGTRAKYQRQLARAIKRARYLSLLPYTDRHQ

E-mail results to
Once the job is finished, the results will be sent to the E-mail address provided in this field. It is strongly recommended to provide E-mail address, as the E-mail message contains the link to the results of the finished job.

Job description (max 20 characters)
A user may provide an optional job identifier of maximum length of 20 characters.

Construction of multiple sequence alignment using PSI-BLAST

Do not run PSI-BLAST (use my input multiple alignment)
Providing a custom multiple sequence alignment as an input and checking this option disallows PSI-BLAST (Altschul SF, et al., 1997, Nucleic Acids Res., 25:3389-402) run using the provided multiple sequence alignment for jumpstart; the input alignment is used directly to construct query profile.

Let the server handle PSI-BLAST run
Selecting this option means that the server will take control of the process of profile construction.

Server output
Sequence or MSA
A file that contains an input sequence or an input multiple sequence alignment.

Parameters
A file that contains an input parameters for the server run.

STDERR log, STDOUT log
Output files of the standard output and standard error streams.

Final multiple alignment
A constructed final multiple alignment if PSI-BlAST was used. Otherwise only secondary structure information is added to the input multiple sequence alignment.

Comparison of MSA
An HHsearch (Söding J, 2005, Bioinformatics, 21:951-60) output file.

Aligned PDEXK
A pseudo multiple sequence alignment of the following structure:
  1. >ss_pre, secondary structure sequence predicted by PSIPRED (Jones DT, 1999, J Mol Biol., 292:195-202).
  2. >Consensus, consensus sequence of HHsearch.
  3. >A1W682_ACISJ/1, query sequence (or the first sequence in the provided MSA).
  4. >DELIM1, delimiter line.
  5. >PDEXK_family_pdexk, definition line of the conservative positions of PDEXK_family. 1 - E; 2 - PD; 3 - EXK.
  6. >PDEXK_family, PDEXK protein family aligned with query.
  7. >DELIM2, delimiter line.
  8. Final multiple alignment.
 SVM Probability
A file of SVM (Lin H-T, Li L, 2008, J Mach Learn Res., 9:285-312) probabilities. The file structure is as follows: name of a classifier, its training accuracy and probability.

Probability
[0.8, 1.0]. A query protein family is considered as a member of PDEXK clan.
[0.5, 0.8). Uncertain.
[0.0, 0.5). Unlikely. -1 means that no PDEXK was aligned with the query.

References
